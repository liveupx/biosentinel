## ЁЯза How the AI Works

BioSentinel doesn't look at a single blood test result. It tracks **patterns over time** тАФ the subtle trends that no single checkup can reveal:

| What a Clinician Sees | What BioSentinel Detects |
|---|---|
| HbA1c = 5.9% тАФ "borderline, watch it" | 5.5 тЖТ 5.6 тЖТ 5.7 тЖТ 5.8 тЖТ 5.9 over 24 months = **pre-diabetes trajectory** |
| CEA = 3.2 ng/mL тАФ "within limits" | 1.5 тЖТ 1.9 тЖТ 2.3 тЖТ 2.8 тЖТ 3.2 over 18 months = **rising tumour marker** |
| Lymphocytes = 24% тАФ "low-normal" | 32% тЖТ 29% тЖТ 27% тЖТ 25% тЖТ 24% = **declining immune cells, hematologic alert** |

**Research basis:**
- Nature Medicine (2023): AI on 6M records predicts pancreatic cancer **36 months in advance**, AUROC 0.88
- Google/NIH: CNN on 42,290 CT scans achieves AUC 95.5%, outperforms average radiologist
- BMC Medical Research (2025): Longitudinal EHR models consistently outperform single-visit baselines

## ЁЯЪА How to Run

### Option 1 тАФ Local (Recommended for getting started)

#### Prerequisites
- Python 3.10 or higher
- Mac: `brew install python@3.11` тЖТ then use `python3.11`
- Windows: download from [python.org](https://python.org/downloads) тАФ check тЬЕ "Add to PATH"
- Linux: `sudo apt install python3.11 python3.11-pip`

#### Step 1 тАФ Install packages (one time only)

**Mac/Linux:**
```bash
pip3.11 install fastapi "uvicorn[standard]" sqlalchemy pydantic \
  "python-jose[cryptography]" "passlib[bcrypt]" \
  scikit-learn numpy python-multipart
```

**Windows:**
```bash
pip install fastapi "uvicorn[standard]" sqlalchemy pydantic ^
  "python-jose[cryptography]" "passlib[bcrypt]" ^
  scikit-learn numpy python-multipart
```

#### Step 2 тАФ Start the server

**Mac/Linux:**
```bash
python3.11 run.py
# or directly:
python3.11 app.py
```

**Windows:**
```bash
# Double-click START_WINDOWS.bat
# or:
python run.py
```

#### Step 3 тАФ Open the dashboard

Open **`biosentinel_dashboard.html`** in Chrome, Firefox, or Safari.

### Option 4 тАФ Configure Email Alerts

```bash

```

Edit `.env`:
```env
EMAIL_ENABLED=true
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your-gmail@gmail.com
EMAIL_PASS=xxxx-xxxx-xxxx-xxxx    # Gmail App Password (not your login password)

EMAIL_TO_ADMIN=doctor@hospital.com
```

**Gmail App Password setup:**
1. Google Account тЖТ Security тЖТ 2-Step Verification тЖТ turn ON
2. Google Account тЖТ Security тЖТ App Passwords
3. Select "Mail" тЖТ Generate тЖТ copy 16-character code
4. Paste into `EMAIL_PASS` above

BioSentinel will then automatically email you when a patient prediction reaches HIGH or CRITICAL risk.

## ЁЯФР Security & Privacy

- All patient data stored **locally only** in `biosentinel.db` тАФ never sent to external servers
- Multi-user isolation enforced at API level тАФ HTTP 403 on cross-user access attempts
- JWT tokens expire after 24 hours
- Passwords hashed with bcrypt (SHA-256 fallback)
- Audit log captures every patient data access and AI prediction
- Email passwords stored in the database (encrypt in production тАФ see `.env.example`)
- SQLite file should be backed up and encrypted at rest (VeraCrypt or similar)
- **For HIPAA/GDPR compliance** тАФ additional steps required: PostgreSQL + full encryption + access controls + data processing agreements

## ЁЯЪз Known Limitations (Be Honest)

1. **Models trained on synthetic data** тАФ Risk scores are directionally correct but not clinically validated. A patient scoring 59% cancer risk does NOT mean they have a 59% chance of getting cancer тАФ it means their longitudinal biomarker pattern resembles patterns associated with cancer risk in the training data. MIMIC-IV retraining is in progress.

2. **Lab report OCR accuracy** тАФ The Claude Vision OCR pipeline works on standard digital PDF lab reports. Heavily degraded scans, unusual fonts, or multi-column formats may extract incorrect values. Always verify extracted values before saving.

3. **SQLite for single-server only** тАФ Fine for a clinic with one server. For multi-location or cloud deployment, use the `--profile postgres` flag in docker-compose. See `.env.example` for PostgreSQL setup.

4. **English only** тАФ All UI and alerts are in English. Multi-language support is on the roadmap.

5. **Predictions cluster at extremes** тАФ Due to clean separation in synthetic training data, scores tend toward ~5% (healthy) or ~35-60% (at-risk) rather than a smooth distribution. Real clinical data will produce more graduated scores.

6. **Claude AI requires API key** тАФ Set `ANTHROPIC_API_KEY` in `.env` to enable Vision OCR, narrative generation, and anomaly detection. All features degrade gracefully without it.
